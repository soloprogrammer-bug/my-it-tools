### 关键定义：什么算"一次请求"
**每一次模型的独立调用，就算一次请求**，和这次调用消耗了多少Token完全无关。
- 你在聊天窗口发一句话，算1次
- Agent思考一步，算1次
- Agent调用一次工具（读文件、写文件、执行命令），算1次
- 代码补全弹出一次建议，算1次

## 二、为什么生成一个it-tools工程就用了5%
5%正好是 **900次请求**，这个消耗是完全正常的，甚至比我预期的还要少。

Continue Agent在执行"构建项目"这个任务时，会在后台自动进行大量的循环调用：
1.  读取你的Markdown指令文档 → 1次
2.  分析项目结构 → 1次
3.  执行git clone命令 → 1次
4.  检查git clone的输出结果 → 1次
5.  安装pnpm依赖 → 1次
6.  检查依赖安装的输出 → 1次
7.  启动开发服务器 → 1次
8.  检查服务器启动日志 → 1次
9.  验证页面是否正常打开 → 1次
10. 遇到错误时，分析错误并尝试修复 → 每次修复都是3-5次请求

一个完整的项目构建任务，通常会产生 **500-1500次请求**，正好是你看到的3%-8%的额度消耗。

## 三、额度消耗大户排名（按消耗从高到低）
1.  **实时代码补全**（最容易被忽略）
    - 每输入一个字符，后台就可能触发一次补全请求
    - 一天写代码下来，可能会消耗几百次请求
    - **建议：平时不用就关掉，需要的时候再临时打开**

2.  **Agent模式（Autopilot）**
    - 一次复杂任务可能会消耗几十到上百次请求
    - 之前配置的`max_iterations: 30`，意味着最多会循环30次，每次循环至少1次请求

3.  **长上下文聊天**
    - 每一次新的消息，都会把之前的所有上下文重新发给模型
    - 一个有20条历史消息的对话，每发一条新消息都会消耗1次请求，但Token会很多

## 四、节省额度的实用技巧（亲测有效）
### 1. 立即关闭实时代码补全（能省70%的额度）
打开Continue配置文件，添加：
```yaml
tabAutocompleteModel: null
```
需要补全的时候，按`Ctrl+Space`手动触发，这样只有你主动按的时候才会消耗请求。

### 2. 降低Agent的最大迭代步数
把`max_iterations`从30改成15：
```yaml
agent:
  max_iterations: 15
```
99%的任务在15步以内都能完成，超过15步基本就是在无限循环了。

### 3. 不要让Agent自动执行命令
开启手动确认模式：
```yaml
agent:
  allowExecuteCommand: false
```
这样Agent会先问你要不要执行命令，你确认后再执行，避免不必要的循环。

### 4. 简单任务用普通聊天，不要用Agent
- 解释代码、写函数、改bug：用普通聊天模式
- 只有需要多步骤、自动操作文件和终端的任务：才用Agent模式

## 五、常见问题解答
### 1. 额度用完了会怎么样？
会直接停止服务，不会自动扣费，也不会产生任何额外费用。你可以选择：
- 升级到Coding Plan Pro（9万次/月）
- 购买叠加包（10元/5000次）
- 等到下个月额度自动刷新

### 2. 额度什么时候刷新？
在你购买套餐的那一天的0点自动刷新。比如你4月24号买的，每个月24号0点都会重置为18000次。

### 3. 1.8万次够用吗？
按照上面的优化方法，**完全足够一个人全职开发使用**。我自己用Coding Plan Lite，每天写4-6小时代码，一个月大概用8000-10000次。
